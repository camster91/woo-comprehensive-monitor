# Woo Monitor v3.1.0 — Project Completion Design

**Goal:** Complete all remaining features, production config, dashboard polish, and DevOps for the woo-comprehensive-monitor project.

**Base:** v3.0.0 (React + SQLite rebuild, deployed at woo.ashbi.ca)

---

## 1. Production Config & Missing Endpoints

- Configure Mailgun (MAILGUN_API_KEY, MAILGUN_DOMAIN, ALERT_EMAIL) on live container
- Enable dashboard auth (REQUIRE_AUTH=true, ALLOWED_EMAILS)
- Set DeepSeek API key for AI chat
- Add `GET /download/plugin` route serving latest plugin ZIP from `plugin-releases/` directory
- Fix plugin version mismatch: WCM_VERSION constant 4.5.1 → 4.5.2

## 2. Slack Alerts

- New `slack-service.js` — POST to Slack webhook URL for critical/high severity alerts
- Configurable via `SLACK_WEBHOOK_URL` env var
- Rich message formatting with severity colors, store name, alert details

## 3. Real-Time Dashboard (SSE)

- Server: `GET /api/events` SSE endpoint with in-memory event emitter
- Events: `alert:new`, `store:updated`, `health:complete`
- Client: `useEvents` React hook, auto-reconnect on disconnect, fallback to 60s polling
- Toast notification system for real-time alert popups and action feedback

## 4. Browser Console Capture

- Plugin: Extend error-tracker to capture `console.error` with stack traces
- Bundle last 10 console entries, send as `console_logs` field in error payload
- Server: `console_logs` table linked to alerts via alert_id
- Dashboard: Console logs shown inline when expanding an alert

## 5. Performance Metrics (Web Vitals)

- Plugin: Lightweight Web Vitals collector (LCP, FID, CLS, TTFB)
- Sends `type: "performance_metrics"` to `/api/track-woo-error`, once per page load
- Server: Store in `performance_metrics` table, aggregate daily averages in store_stats
- New `GET /api/stores/:id/performance` endpoint
- Dashboard: Performance tab on Overview with Web Vitals charts per store

## 6. Dashboard Polish

- React error boundaries on every page (crash isolation + retry button)
- Audit logging: `audit_log` table, middleware logs all non-GET API calls
- New "Audit Log" page with filtering by action type and user
- Plugin download button on Stores page, version info on System page
- Toast notification system (replaces inline messages)

## 7. DevOps — GitHub Actions CI/CD

- On push to master: install deps, build dashboard, verify server boots
- CD step: SSH into server, pull, rebuild Docker image, swap container
- Repository secrets for SSH key and env vars

## 8. Database Backups

- `backup-db.js` script: exports SQLite, commits to `backups` branch on GitHub
- Runs via node-cron at 2am daily
- Keeps last 30 backups, auto-prunes older ones

---

## New Database Tables

- `console_logs` (id, alert_id, entries JSON, timestamp)
- `audit_log` (id, action, user, target, details JSON, timestamp)
- `performance_metrics` (id, store_id, page_url, lcp, fid, cls, ttfb, timestamp)

## New Services

- `slack-service.js` — Slack webhook integration
- `backup-db.js` — Daily DB backup to git
- SSE event emitter (in alert-service / tracking route)

## Plugin Changes

- Console error capture in error-tracker
- Web Vitals collector
- Version bump to 4.5.2

## Decisions

- SSE over WebSockets (simpler, one-way push is sufficient)
- Slack only (no SMS/Discord)
- GitHub Actions for CI/CD (not relying on Coolify)
- Daily DB backup to GitHub backups branch (not S3)
- Keep email-only auth (no TOTP — email codes are sufficient)
- Page load metrics only (no API timing)
- Full audit log (all admin actions, not just security)
- Console logs attached to alerts (not standalone)
